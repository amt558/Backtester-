Build order — dependency-ordered, ~8 days + §1 fix
  1. Slice -1 — §1 confound resolution. Pick (A)/(B)/(C) from Confound panel above. Cheapest is (C) retrospective on trades.csv — recon §7 estimated 1 session. Without this, every downstream stat is contaminated. Hard prerequisite for Slice 6.
  2. Slice 0 — Ledger schema extension (½ day). Add signal_values_json, thresholds_json, accepted_bool, reject_reason to existing data/tradelab_history.db::runs. Backfill from existing reports/*/robustness_result.json. Prerequisite for Slice 6.
  3. Slice 0.5 — Engine Integrity / Canary panel (½ day). 4 canary strategies + dashboard banner + Accept-block when any canary mismatches. Doesn't depend on §1.
  4. Slice 1a — Hold-out as gate (½ day). FULL spec §4.2. Orthogonal failure mode (data leakage) — doesn't depend on §1 or calibration evidence.
  5. Slice 1b — Relative context section in Score modal (½ day). FULL spec §4.7. Pure frontend.
  6. Slice 2 — Multi-dim correlation gate (1.5 days). FULL spec §4.3. Independent of §1 — computed from live trade timestamps + PnL, not verdicts.
  7. Slice 3 — Outcome backfill cron + per-signal hit rate (1 day). User's Slices C+D. Daily batch over the extended ledger. Depends on Slice 0 + Slice -1.
  8. Slice 4 — Regime banner only (1 day). LITE behavior. Per-strategy regime fit deferred until Slice 3 evidence shows the classifier predicts live performance.
  9. Slice 5 — Live divergence (K-S + decay) as REVIEW only (1 day). LITE behavior. Auto-disable upgrade gated on Slice 3 evidence showing K-S correlates with real failure.
  10. Slice 6 — Verdict Accuracy Loop banner (1 day). Combines portfolio stats (FULL §4.5) with per-signal hit-rate panel (Slice 3). Hit-rate tags inline on Score modal diagnostic cells. Depends on Slice -1 + Slice 0 + Slice 3.
Net: ~8 days of build + 1-5 days for §1 resolution depending on path chosen.
Compared to current spec (6 days): +2 days build + the §1 cost. Buys: actual feedback loop, canary integrity, honest calibration. Saves: not shipping K-S auto-disable on uncalibrated test, not shipping per-strategy regime fit on thin samples — both of which would need to be ripped out later when ledger evidence shows them unreliable.
The honest read — when to pick which version
Pick FULL if: you've already resolved §1 (deployed code matches scored code), you have ≥60 days of regime-tagged outcome data validating the classifier, and you trust K-S on 30 trades enough to give it kill-switch authority. None of those preconditions hold today. Don't pick FULL.

Pick LITE if: you want a smaller build that softens the two riskiest pieces of FULL but accepts that the calibration banner remains a 3-stat side-show rather than a real feedback loop, and you're willing to leave §1 contamination invisible. Better than FULL but still missing the centerpiece.

Pick CALIBRATED if: you want to honor the recon §7 recommendation explicitly — close the verdict-accuracy loop first, surface the §1 confound that contaminates everything else, and add only the gates whose value is independent of calibration evidence (hold-out, multi-dim correlation). Defer K-S auto-disable and per-strategy regime fit until ledger evidence justifies them. This is what the recon was asking for.

Practical sequence: ship Slices -1 → 0 → 0.5 → 1a → 1b → 2 first (~4 days + §1 cost). That's the orthogonal-coverage portion plus the integrity surfaces — value lands quickly. Then Slices 3 → 4 → 5 → 6 over the next ~4 days, with the calibration banner becoming meaningful as the ledger fills.
